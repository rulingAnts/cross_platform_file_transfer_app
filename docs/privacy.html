Privacy Policy

Last updated: January 2024

Rapid Transfer collects ZERO user data. All file transfers happen directly between your devices on your local network. We don't have servers, we don't track you, and we never see your files.

Overview

Rapid Transfer is an open-source, privacy-first file transfer application. This privacy policy explains how the application works and what data it does (and doesn't) collect.

Data Collection

What We DON'T Collect

  • Personal Information: No names, emails, phone numbers, or accounts required
  • File Content: Your files never touch our servers (we don't have any!)
  • Usage Analytics: No tracking, no telemetry, no analytics
  • Crash Reports: No automatic error reporting
  • Device Information: No collection of device specs or identifiers
  • Location Data: No GPS or location tracking
  • Network Activity: No logging of transfers or connections

What Data Stays on Your Device

The following data is stored locally on your device and never transmitted to us:

  • Device Name: Your chosen device name (stored locally)
  • Trusted Devices: List of devices you've previously connected to (stored locally)
  • Transfer History: Records of past transfers (stored locally, can be cleared)
  • App Settings: Your preferences (language, stream count, etc.)
  • TLS Certificates: Device certificates for secure connections (stored locally)

How Transfers Work

Direct Device-to-Device

All file transfers happen directly between your devices on your local network:

  • Devices discover each other using UDP broadcast (standard local network protocol)
  • Files are encrypted with TLS 1.3 and sent directly between devices
  • No internet connection required
  • Files never pass through any third-party servers

Cloud Relay (Optional)

If you choose to set up the optional cloud relay feature:

  • You control the relay server (self-hosted or your chosen provider)
  • Files are end-to-end encrypted before being uploaded
  • Files are automatically deleted after download
  • We have no access to your relay server or files

Network Permissions

Rapid Transfer requires the following permissions to function:

  • Network Access: To discover devices and transfer files
  • Storage (Android): To read and write files you want to transfer
  • Location (Android): Required by Android for WiFi Direct and hotspot features (not used for actual location tracking)
  • Notifications: To notify you when transfers complete

Open Source

Rapid Transfer is 100% open source (MIT License). You can:

  • Review the source code on GitHub
  • Verify that no data collection occurs
  • Build the app yourself from source
  • Audit the security and privacy features

Third-Party Services

Rapid Transfer does not use any third-party analytics, advertising, or tracking services. The only external connections are:

  • GitHub (if you download updates - optional)
  • Your self-hosted cloud relay server (if you configure it - optional)

Security

Encryption

  • All transfers use TLS 1.3 encryption (military-grade)
  • Certificate pinning prevents man-in-the-middle attacks
  • SHA-256 checksums verify data integrity

Local Data Storage

  • Settings and history stored in standard OS-protected locations
  • Desktop: ~/.rapidtransfer/ (protected by user permissions)
  • Mobile: App-specific storage (protected by Android sandbox)

Children's Privacy

Rapid Transfer does not collect any data from anyone, including children under 13. The app can be safely used by users of all ages.

Changes to This Policy

Since we don't collect data, this policy is unlikely to change. Any updates will be posted on this page and in the app's GitHub repository.

Contact

For questions about privacy or security, please:

  • Open an issue on GitHub
  • Review the source code for verification
